# Operating Rule — Definition of Productized

No OperOS asset is considered productized until it has:

1. Source object ID
2. Canonical GitHub path (this repo) or approved interim Confluence source
3. Confluence portal page
4. Jira work item
5. Version history
6. Visual model
7. Reusable output path
8. Sales translation
9. AI augmentation and automation prompt
10. Databricks ingestion and evaluation state
11. QA review
12. Release note entry

No framework, playbook, sales asset, or prompt is Published until its result metric is defined in RoT terms (see `core-concepts/RoT-return-on-tokens.md`).
