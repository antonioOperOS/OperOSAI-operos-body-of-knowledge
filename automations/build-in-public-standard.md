# Build-in-Public Content Standard

Every daily framework migration produces three post drafts against this standard. The bar: would a smart, skeptical operator stop scrolling and actually read it — not "would this pass as a LinkedIn post."

## Non-negotiables

1. **One real detail per post, not a summary.** Pull one specific, concrete thing out of that day's actual framework — a failure mode, a metric, a tension between two ideas, a decision rule — and build the post around that one thing. Never "today we published our Leadership framework." Always the one sharp idea inside it.
2. **Show the work, not the announcement.** Per the trust-first principle already governing Sales Enablement: buyers and operators are skeptical of polish. A real tension, a real tradeoff, a mistake avoided — beats "excited to share."
3. **No corporate throat-clearing.** Banned openers: "Excited to announce," "Thrilled to share," "In today's fast-paced world," "Game-changer," "Unlock," "Leverage," "Synergy," "Journey" (as a noun about the company). If a sentence could appear in any startup's post unchanged, cut it.
4. **Specificity beats adjectives.** Not "a powerful framework" — the actual mechanism, the actual number, the actual named failure mode from the doc. Adjectives are a tell that the substance is missing.
5. **RoT-anchored where it's genuinely relevant.** If the framework has a real outcome/metric angle, close the loop in RoT terms (revenue per token vs. holdout) — but never force it in artificially; a post about, say, MECE issue trees doesn't need RoT bolted on.
6. **End on a real artifact or a real question** — a link to the actual file, a genuine either/or choice the reader has to make, or a number — never a generic CTA ("thoughts?", "let's connect").
7. **Length:** 60–120 words for LinkedIn. If it needs more, it's two posts, not one long one.

## The three voices — genuinely different, not reworded copies

- **Antonio (builder/founder voice):** First person, present tense, a little raw. What it was like to actually build this, what surprised him, what he changed his mind about. Never sounds like marketing copy.
- **Rafael (rigor/metric voice):** Leads with a number, a mechanism, or a measurement discipline. Slightly more technical, closer to how he already talks about RoT. Skeptical of hype, proves things.
- **OperOS company voice:** Third-person-adjacent but not corporate — positions OperOS's own operating discipline as the proof point ("we run our own company on this before we sell it"), per the dogfooding thesis already established. Never sales-y.

## Before logging a draft, check it against this list

- Does it name one specific real thing from today's work? (Not "we made progress.")
- Could this exact post have been written about a company that hadn't built anything? If yes, rewrite.
- Is any sentence replaceable with corporate boilerplate and still read the same? If yes, cut it.
- Does each of the three versions actually sound like a different person, or did I just swap a few words?
