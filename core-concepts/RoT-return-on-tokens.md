# RoT — Return on Tokens

**Status:** Canonical (v3, approved). This is the OperOS signature metric and must be referenced as foundational knowledge in every framework, playbook, sales asset, and prompt produced under this Method.

## Definition

RoT (Retorno sobre Tokens / Return on Tokens) is the outcome metric measuring revenue generated per token spent, proven against a holdout — not activity, not token consumption. It is the signature metric of OperOS's "outcome layer" positioning: OperOS is the vendor-neutral layer above a client's AI stack that runs governed agentic loops and measures incremental revenue against holdout.

## Core components

- **Outcome instrumentation** — the result metric is defined before shipping any governed agentic loop.
- **Measurement against holdout** — proving incrementality, not correlation.
- **Governance** — auditability, human-in-the-loop, and kill switch as B2B contract prerequisites.
- **Vendor-neutral orchestration** — across the client's existing models, agents, and data.

## Canonical vocabulary (use consistently — no synonyms)

RoT, outcome layer / camada de resultado, governed agentic loop, holdout, incremental revenue.

## Integration rule

No framework, playbook, sales asset, or prompt is considered Published until its result metric is defined in RoT terms (revenue per token, measured against holdout).
